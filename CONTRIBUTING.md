# Участие в разработке (Contributing)

Этот проект является общедоступным для образовательных и справочных целей.

⚠️ Он не поддерживается как community-driven open-source проект 
и не предполагает внешней разработки, расширения функциональности или принятия сторонних pull requests.

---

## Статус проекта

Проект является завершённым справочным / образовательным проектом 
демонстрирующий реализацию управляемых JWT-сессий на Django, DRF, PyJWT и Redis.

⚠️ Активная разработка новых функций не планируется.

ℹ️ Возможны только:
- исправления воспроизводимых ошибок;
- уточнения документации;
- технические правки, не меняющие основную архитектуру проекта.

Архитектура, API и поведение JWT-flow считаются зафиксированными в рамках текущей версии проекта.

---

## Issues (ошибки)

__GitHub Issues__ используются исключительно для сообщений об ошибках.

Перед созданием Issue:
- проверьте, нет ли уже похожей проблемы в существующих Issue;
- убедитесь, что проблема воспроизводится на актуальной версии проекта;
- подготовьте минимальные шаги для воспроизведения.

Создайте [Issues](https://github.com/NikPACodes/JWTGuard/issues) типа __"Сообщение об ошибке"__ и укажите:
- краткое описание ошибки;
- шаги для воспроизведения;
- ожидаемое поведение;
- фактическое поведение;
- логи, traceback или текст ошибки;
- сведения об окружении: ОС, версия Python, версия Django, способ запуска;
- способ запуска проекта: Docker или локальное окружение.


⚠️ Важно: вопросы безопасности __не публикуйте в Issues__ — используйте [SECURITY](./SECURITY.md).

---

## Pull Requests (внесение изменений в код)

Репозиторий опубликован только для просмотра, изучения архитектуры и воспроизведения проекта локально.

⚠️ Сторонние pull requests, включая исправления, рефакторинг, улучшения документации и предложения по архитектуре, 
__могут быть закрыты без review__.

---

## Запрос функциональности (Feature request)

⚠️ Запросы на добавление новых функций __не принимаются__.

Проект не планируется развивать как продукт или библиотеку общего назначения.
